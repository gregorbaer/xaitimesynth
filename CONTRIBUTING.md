# Contributing to xaitimesynth

Thank you for your interest in contributing to xaitimesynth!

## Getting Started

See the [Development Setup](https://gregorbaer.github.io/xaitimesynth/contributing/development-setup/) guide for instructions on setting up your local environment.

## Reporting Issues

Please open an issue on [GitHub](https://github.com/gregorbaer/xaitimesynth/issues) with a clear description of the problem and steps to reproduce it.

## Pull Requests

1. Fork the repository and create a branch from `dev`.
2. Make your changes, including tests where appropriate.
3. Ensure all tests pass (`make test`) and linting is clean (`make lint`).
4. Open a pull request against `dev` with a clear description of the changes.

## Code Style

This project uses [Ruff](https://docs.astral.sh/ruff/) for linting and formatting. Run `make lint` and `make format` before committing.

## Commit Messages

See the [Commit Guidelines](https://gregorbaer.github.io/xaitimesynth/contributing/commit-guidelines/) for conventions used in this project.
